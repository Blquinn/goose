Better tests with containers

Managing state is hard. Managing database state is even harder. And coordinating state in a test suite is just a bad time.

But it doesn't have to be this way!

There is a fantastic Go package called ory/dockertest that enables you to spin up ephemeral docker containers. It will work locally (assuming Docker is installed) and in your Continuous Integration (CI) pipelines.

After applying thousands of migrations in production we know goose is production-ready and just works. But we have plans to add more functionality to goose, and integration tests are a welcome addition to the goose test-suite.

In a recent PR#276 we added container-based database tests. These tests spin up a fresh database per test. Yes, that's right, we're talking dozens of containers.

Getting started is easy, here's a quick example:

options := &dockertest.RunOptions{
    Repository: "postgres",
    Tag:        "14-alpine",
    Env: []string{
        "POSTGRES_USER=postgres",
        "POSTGRES_PASSWORD=password1",
        "POSTGRES_DB=testdb",
        "listen_addresses = '*'",
    },
    // (1)
    Labels:       map[string]string{"goose_test": "1"}, 
}
container, err := pool.RunWithOptions(
    options,
    func(config *docker.HostConfig) {
        // Set AutoRemove to true so that stopped container goes away by itself.
        config.AutoRemove = true
        config.RestartPolicy = docker.RestartPolicy{Name: "no"}
    },
)

// Fetch port dynamically assigned to container, something like 55,245
dynamicPort := container.GetPort("5432/tcp") 

// Get a *sql.DB connection, using the dynamic port assigned to container
// postgresql://postgres:password1@localhost:{dynamicPort}/testdb?sslmode=disable
db, err := sql.Open(dialectPostgres, psqlInfo)

  1. To simplify cleaning up containers. In TestMain we added a flag -debug to disable automatic container cleanup. This allows connecting to the container and poking around the database. Bonus, we added -port flag so we don't have to figure out the random port and can maintain an open connection with tools such as Table Plus.

    go test -v -count=1 ./tests/e2e -run=TestMigrateFull -port=5432 -debug

    Sometimes there might be a bunch of containers, but one can easily clean up all containers by leveraging the container label:

    docker stop -t=0 $(docker ps --filter="label=goose_test" -aq)

After each test has run the container is cleaned up, something like this:

t.Cleanup(func() {
    if err := pool.Purge(container); err != nil {
        log.Printf("failed to purge resource: %v", err)
    }
})

For integration tests this is perfect. We can spin up a fresh lightweight container for each test, and not worry about tests stomping on each other or having to coordinate state between them.

The entire thing is fast! Most tests run in parallel t.Parallel() and the entire integration test-suite runs in about 6-7s (for postgres:14-alpine).
